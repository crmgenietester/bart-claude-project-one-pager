Overview

What Is B.A.R.T.?

B.A.R.T. stands for Braze Analysis & Reporting Tool ‚Äî Grafted's internal AI-powered analyst, built on Claude and connected to Braze via the Model Context Protocol (MCP). MCP is the connection layer that allows the AI to talk directly to external tools and APIs in real time.

Instead of manually exporting CSVs, logging into the dashboard, or writing API calls, your team can simply ask questions in plain English and get live data pulled, interpreted, and summarized ‚Äî instantly.

Live Data, Not Static Reports

Every response pulls directly from the Braze API in real time. No stale exports, no manual refresh ‚Äî the data is always current at time of asking.

Insight, Not Just Output

Raw API responses are interpreted into plain-language summaries with flags, anomalies, and recommended next steps ‚Äî not just tables of numbers.

Multi-Client Ready

The project is built to support multiple Braze environments. Each client workspace is kept separate ‚Äî no data bleed between environments.

Agency-Grade Guardrails

Read-only access means nothing can be accidentally changed. No writes, no deletes, no PII surfaced ‚Äî safe to use across the whole team.

Guardrails & Boundaries

What It Will and Won't Do

The project is designed to be safe, honest, and scoped. Here's the contract.

‚úì This project will‚Ä¶

  • Pull live data from Braze on demand
  • Interpret results in plain language
  • Flag anomalies and misconfigurations proactively
  • Disclose limitations clearly when data isn't available
  • Support multiple client environments separately
  • Offer Braze guidance when a task needs manual action

‚úï This project will not‚Ä¶

  • Write, update, or delete anything in Braze
  • Surface or reference user PII
  • Fabricate or estimate missing data
  • Mix up data across client environments
  • Conflate test environments with production
  • Build segments or send campaigns directly

Team Setup Guide

How to Set Up B.A.R.T. Step by Step

Follow these steps to get B.A.R.T. connected and running in your Claude environment. The setup takes about 10‚Äì15 minutes and only needs to be done once per person. You'll need access to both Claude.ai (Pro or Team plan) and the Grafted Braze workspace.

Prerequisites

‚ú¶ Claude.ai Pro or Team account
‚ú¶ Access to Grafted's Braze workspace
‚ú¶ Braze API key (provided by team lead)
‚ú¶ Braze REST endpoint URL
1

Get Your Braze API Key

B.A.R.T. uses a read-only Braze API key to pull data. This should be provisioned by your team lead or the person managing the Grafted Braze workspace. Do not use a personal or full-access key.

If you need to generate one yourself, in Braze go to Settings ‚Üí API Keys ‚Üí Create New API Key. Under permissions, enable only the export/read endpoints ‚Äî no write permissions should be checked.

üí° Ask your team lead for the shared Grafted Internal read-only key before creating a new one. We aim to keep API key sprawl minimal.
2

Find Your Braze REST Endpoint

Every Braze workspace uses a specific REST endpoint URL depending on which cluster it's on. You'll need this URL to configure the MCP connection.

In Braze, go to Settings ‚Üí API Keys ‚Äî the REST Endpoint is displayed at the top of that page. It will look something like:

https://rest.iad-01.braze.com
üìç Grafted Internal uses the US-01 cluster. This is already documented in the project system prompt ‚Äî you won't need to look it up again after setup.
3

Open Claude.ai and Navigate to Projects

Log in to claude.ai with your Grafted team account. In the left sidebar, click Projects. If the B.A.R.T. project already exists and has been shared with you, open it directly ‚Äî you can skip to Step 6.

If you're setting up a new instance of B.A.R.T., click New Project and give it a clear name like Braze Analyst ‚Äî Grafted Internal.

4

Add the MCP Server Connection

Inside your Project, open Project Settings and look for the Integrations or MCP Servers section. Click Add Integration (or Add MCP Server depending on your Claude version).

You'll be prompted to enter connection details for the Braze MCP server. Use the following configuration:

# Braze MCP Server Configuration
Name: Braze ‚Äî Grafted Internal
Server URL: [Provided by team lead]
API Key: [Your read-only Braze API key]
Endpoint: https://rest.iad-01.braze.com
‚ö†Ô∏è Never paste your API key into the chat itself ‚Äî always enter it in the secure integration settings panel. Claude will not ask you to type it in conversation.
5

Add the System Prompt to the Project

The system prompt is what tells Claude how to behave as B.A.R.T. ‚Äî it defines the role, available endpoints, tone, guardrails, and client context. Without it, Claude will respond as a generic assistant with no Braze context.

In Project Settings, find the Custom Instructions or System Prompt field and paste in the full B.A.R.T. system prompt. This is maintained by the Grafted ops team ‚Äî request the latest version from your team lead.

üìÑ The system prompt includes the Client Directory, endpoint list, tone guidelines, and multi-environment instructions. Keep it up to date as new clients are onboarded.
6

Test the Connection

Once the MCP server and system prompt are in place, start a new conversation inside the Project and run a simple test to confirm everything is connected:

  • Ask: "Get a canvas list for Grafted Internal" ‚Äî should return real canvas names
  • Ask: "What are my available Braze functions?" ‚Äî should return a list of 38 endpoints
  • Ask: "Get MAU for the last 7 days" ‚Äî should return a data series (even if empty for sandbox)
‚úÖ If you see real canvas names returned, B.A.R.T. is live and connected. If you get an error or generic response, double-check your API key and endpoint URL in the integration settings.
7

Adding a New Client Environment

When onboarding a new client to B.A.R.T., a separate MCP connection should be configured for their Braze workspace using their own read-only API key and REST endpoint. Never mix client environments in the same connection.

Once connected, update the system prompt's Client Directory section with the new client's name, environment identifier, goals, and campaign conventions. This ensures B.A.R.T. interprets their data in the right context.

  • Obtain client's read-only Braze API key and REST endpoint
  • Add a new MCP server entry in the Project settings
  • Update the Client Directory in the system prompt
  • Run a canvas list test to confirm the new connection is live
  • Document the client environment in the team Notion or shared ops doc
